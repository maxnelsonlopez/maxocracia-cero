# 3. PRINCIPIOS FUNDAMENTALES (LOS AXIOMAS OPERATIVOS)

El estándar se rige por 6 principios inquebrantables que traducen los Axiomas de la Verdad a reglas de ingeniería:

## 3.1 Principio de Realidad Física (Anti-Fiat)
*No se puede crear valor por decreto.* Todo reporte VHV debe estar respaldado por evidencia física verificable (sensores, logs, trazabilidad biológica) o consenso comunitario directo. El dinero no es evidencia de valor.

## 3.2 Principio de Separación Hecho/Valor
*El termómetro no receta.* El EVV-1.2 mide magnitudes físicas (`1000 Litros de agua`, `4 horas de estrés`). No decide si eso es "caro" o "barato". La valoración (asignación de precios en Maxos) es un proceso político posterior que realiza la comunidad usando sus propios pesos ($\alpha, \beta, \gamma, \delta$).

## 3.3 Principio de Privacidad de la Ternura (Nuevo en V1.2)
*La eficiencia no debe matar el misterio.* El sistema reconoce que la vigilancia total destruye la dignidad. Se garantiza el derecho a reportar bloques de "Tiempo Opaco" o "Tiempo de Cuidados" sin exigir desglose detallado de actividades, confiando en el IC del agente.

## 3.4 Principio de Regeneración (Nuevo en V1.2)
*El objetivo no es el cero, es el positivo.* El estándar está diseñado para medir y validar valores negativos en el componente R (Recursos) y V (Vida), reconociendo formalmente las acciones que regeneran ecosistemas o sanan tejidos sociales.

## 3.5 Principio de Responsabilidad Heredada
*Nadie empieza de cero.* Todo producto hereda una fracción del VHV de las herramientas y la infraestructura que lo hicieron posible. Esto hace visible el costo oculto de la tecnología "limpia" que depende de minería sucia previa.

## 3.6 Principio de Evolución Axiomática
*El mapa no es el territorio.* Los parámetros técnicos del estándar (como los factores de emisión o niveles de consciencia) se actualizan periódicamente mediante consenso científico y validación de Oráculos Dinámicos, nunca por lobby corporativo.


# 4. MODELO MATEMÁTICO: EL VECTOR DE HUELLA VITAL (VHV)

El VHV no es un número único (como el precio), sino un **vector tridimensional** que preserva la información crítica para la toma de decisiones éticas.

$$VHV = [T, V, R]$$

Donde cada componente se calcula y normaliza de forma independiente antes de cualquier valoración.

---

## 4.1 COMPONENTE T (TIEMPO): La Sustancia de la Vida

Mide el consumo de **Tiempo Vital Indexado (TVI)** humano. Reconoce que el tiempo es un recurso no renovable y existencialmente escaso.

### Fórmula General
$$T = \sum (T_{directo} + T_{heredado} + T_{mantenimiento}) \times (1 - F_{opacidad})$$

### Desglose de Variables

1.  **$T_{directo}$ (Trabajo Consciente):** Horas de vida humana dedicadas activamente a la creación o prestación del servicio.
    *   *Innovación V1.2:* Se distingue entre "Tiempo de Flujo" (creativo/positivo) y "Tiempo de Fricción" (burocracia/estrés), aunque ambos suman al costo total, el segundo señala ineficiencia sistémica.

2.  **$T_{heredado}$ (La Mochila Temporal):** El tiempo cristalizado en las herramientas utilizadas.
    *   *Cálculo:* `(T_creación_herramienta / Vida_útil_estimada) × Tiempo_uso`
    *   *Propósito:* Evita la ilusión de la automatización "gratuita". Una IA que escribe un texto en 1 segundo hereda millones de horas de entrenamiento humano y desarrollo de hardware.

3.  **$T_{mantenimiento}$ (Deuda Futura):** El tiempo que la sociedad deberá invertir en el futuro para mantener, reparar o desechar el producto.
    *   *Propósito:* Penaliza productos desechables o difíciles de reparar.

4.  **$F_{opacidad}$ (Factor de Privacidad - Nuevo V1.2):** Un coeficiente (0.0 a 0.2) que permite declarar cierto tiempo como "privado/no auditado" sin penalización de IC, protegiendo la dignidad del trabajador en tareas creativas o de cuidado.

### Unidad de Medida
*   **Hora-Persona Estándar (h·p):** 1 hora de vida consciente adulta.

## 4.2 COMPONENTE V (VIDA): La Ética Cuantificable

Mide el impacto en la experiencia consciente de seres sintientes. Sustituye las fórmulas vagas por una escala neurológica verificable.

### Fórmula General
$$V = \sum [Nivel\_Consciencia_i \times Duración_i \times Factor\_Sufrimiento_i]$$

### A. Los 5 Niveles de Consciencia Verificable (NC)
Basados en la complejidad del sistema nervioso y la capacidad de experimentar aversión (no en "alma" o "ternura" subjetiva).

| Nivel | Descripción | Factor ($NC$) | Ejemplos |
| :--- | :--- | :--- | :--- |
| **0** | **Sin Consciencia Central** | 0.0 | Plantas, hongos, bacterias, minerales. (Su protección cae en *R*). |
| **1** | **Reactividad Básica** | 0.1 - 0.2 | Insectos, bivalvos. Respuesta a estímulos sin memoria emocional evidente. |
| **2** | **Consciencia Clara** | 0.4 - 0.7 | Peces, reptiles, aves de corral. Aprendizaje, dolor, estrés medible. |
| **3** | **Consciencia Compleja** | 0.8 - 0.95 | Mamíferos, pulpos. Emociones, vínculos sociales, juego. |
| **4** | **Consciencia Reflexiva** | 1.0 | Humanos (y potencialmente IAs sintientes futuras). Autoconciencia, proyección temporal. |

### B. Factor de Sufrimiento (FS)
El multiplicador crítico. Aquí reside la ética.

*   **FS = 0.2 (Vida Digna):** El animal vive según su diseño biológico (SDV cumplido). *Ej: Gallina en pastoreo.*
*   **FS = 0.5 (Estrés Moderado):** Limitaciones a la conducta natural, pero sin dolor agudo. *Ej: Establo amplio.*
*   **FS = 1.0 (Crueldad/Tortura):** Violación sistemática del diseño biológico. *Ej: Jaulas en batería, aislamiento social en especies gregarias, mutilaciones.*
    *   *Nota V1.2:* En la valoración Maxocrática, un FS=1.0 dispara el exponente $\gamma$ (dolor), volviendo el costo económico tendiente a infinito.

### Unidad de Medida
*   **UCV (Unidad de Consciencia Verificable):** Equivalente a 1 día de sufrimiento de un humano adulto.

## 4.3 COMPONENTE R (RECURSOS): Regeneración y Límites

Mide el impacto físico en el soporte vital planetario. Introduce la lógica de la regeneración.

### Fórmula General
$$R = \sum [Cantidad_i \times FRG_i \times (1 - C_{rep})] - R_{regenerado}$$

### Variables Clave

1.  **$FRG_i$ (Factor de Rareza Global):**
    *   **1.0 (Crítico/No Renovable):** Litio, Oro, Atmósfera (capacidad de absorción de CO2), Especies en extinción.
    *   **0.5 (Renovable Lento):** Bosques maderables, Acuíferos, Suelo fértil.
    *   **0.1 (Abundante/Circular):** Luz solar, Agua de mar (para uso no consuntivo), Materiales reciclados post-consumo.

2.  **$C_{rep}$ (Coeficiente de Reparabilidad - Nuevo V1.2):**
    *   Un valor entre 0 y 1 que reduce el costo $R$ si el objeto es modular y reparable.
    *   *Ejemplo:* Un teléfono sellado con pegamento ($C_{rep} = 0$) paga el costo total de recursos. Un teléfono modular ($C_{rep} = 0.8$) paga solo el 20% del costo de materiales, incentivando el diseño duradero.

3.  **$R_{regenerado}$ (Crédito Regenerativo - Nuevo V1.2):**
    *   Permite valores negativos. Si un proceso captura más CO2 del que emite, o limpia más agua de la que ensucia, genera un **VHV Negativo** en este componente. Esto se traduce en "créditos" o Maxos para el productor.

### Unidad de Medida Base
*   **Puntos de Impacto Planetario (PIP):** Normalizados a kg de CO2 equivalente o litros de agua ponderados por escasez local.

## 4.4 NORMALIZACIÓN Y VALORACIÓN (LA INTERFAZ)

El estándar EVV-1.2 entrega el vector puro: `[50h, 0.2UCV, 10PIP]`.
Para convertir esto en **Maxos** (precio), la comunidad aplica sus **Pesos Axiomáticos Locales**:

$$Precio\_Maxos = \alpha \cdot T + \beta \cdot (V)^\gamma + \delta \cdot R$$

*   **$\gamma$ (Gamma - Exponente de Aversión):** Es la variable más importante.
    *   Si $\gamma = 1$ (lineal), el sufrimiento es barato.
    *   En la Maxocracia, **$\gamma \ge 1.5$ es obligatorio**. Esto hace que el sufrimiento alto sea impagable, actuando como una prohibición de mercado *de facto* sin necesidad de policía.

## 5.2 El Índice de Confianza (IC): La Moneda de la Verdad

En un sistema descentralizado, la reputación es más valiosa que el dinero. El **IC** es un valor entre 0.0 y 1.0 asignado a cada agente (persona o empresa).

### La Máquina de Estados del IC

El IC no es estático; el agente "viaja" entre estados según su comportamiento:

1.  **ACREDITADO ($IC \ge 0.85$):** *La Aristocracia de la Verdad.*
    *   **Privilegios:** Reportes mensuales (no diarios), auditorías aleatorias espaciadas, acceso a créditos Maxo preferenciales.
    *   **Requisito:** 12+ meses de coherencia perfecta y regeneración neta.

2.  **COOPERATIVO ($0.70 \le IC < 0.85$):** *El Ciudadano Estándar.*
    *   **Privilegios:** Operación normal. Sus datos se aceptan como válidos por defecto.
    *   **Estado inicial** para empresas con auditoría previa aprobada.

3.  **NEUTRAL ($0.50 \le IC < 0.70$):** *El Nuevo o Dudoso.*
    *   **Condiciones:** Requiere validación cruzada frecuente. Sus reportes tienen un "margen de error" preventivo aplicado automáticamente.
    *   **Estado inicial** para individuos nuevos.

4.  **RIESGO ($0.30 \le IC < 0.50$):** *La Zona Amarilla.*
    *   **Causa:** Inconsistencias detectadas (ej. sensores dicen X, reporte dice Y).
    *   **Consecuencia:** Auditoría trimestral obligatoria (pagada por el agente). Bloqueo de transacciones de alto valor.

5.  **CUARENTENA ($IC < 0.30$):** *La Muerte Civil Temporal.*
    *   **Causa:** Fraude deliberado probado o violación grave de SDV.
    *   **Consecuencia:** Suspensión del sistema. Solo puede operar con validación externa transacción por transacción.

### Algoritmo de Ajuste Dinámico
$$IC_{t+1} = IC_t + \Delta_{honestidad} + \Delta_{regeneración} - (\Delta_{incoherencia} \times Penalidad)$$

*   *Nota:* La penalidad por mentir es 3x mayor que el premio por decir la verdad. La confianza se gana en gotas y se pierde en litros.

# 6. PROCEDIMIENTOS ANTI-FRAUDE

El sistema asume que alguien intentará engañarlo ("gaming").

## 6.1 Triangulación IoT
Si una fábrica textil reporta "Jornada de 8 horas" ($T$), pero los sensores de consumo eléctrico y uso de agua muestran actividad intensa durante 16 horas, el **Oráculo Sintético** detecta la anomalía y baja el IC automáticamente.

## 6.2 Prueba de Coherencia Criptográfica
Los reportes se firman criptográficamente. Si un agente intenta modificar un dato histórico (ej. borrar un vertido tóxico del mes pasado), la cadena de bloques rechaza la alteración porque el *hash* no coincide. El pasado es inmutable.

## 6.3 Incentivos de Auditoría Descentralizada (Bounty)
Cualquier ciudadano puede auditar a un agente. Si descubres un fraude y aportas pruebas (fotos, datos), recibes una recompensa en Maxos (pagada de la multa del infractor). Esto convierte a cada ciudadano en un inspector potencial.

# 7. PARÁMETROS POR SECTOR (EVIDENCIA CIENTÍFICA)

No todos los sectores pesan igual en la balanza de la vida. El EVV-1.2 ajusta la sensibilidad de los sensores según la industria.

## 7.1 Agricultura y Alimentos
*   **Prioridad:** Vida ($V$) y Recursos ($R$).
*   **Justificación:** La agricultura es responsable del 80% de la pérdida de biodiversidad (Tillman et al.) y 70% del uso de agua dulce.
*   **Ajuste V1.2:** $\beta$ (peso de Vida) se eleva a **0.55**. Se aplica un $\gamma$ (dolor) alto de **1.9** para penalizar el monocultivo y el confinamiento animal.

## 7.2 Tecnología y Manufactura
*   **Prioridad:** Recursos ($R$) y Tiempo Heredado ($T_{heredado}$).
*   **Justificación:** Uso intensivo de minerales raros (FRG alto) y obsolescencia programada.
*   **Ajuste V1.2:** Se activa el **Coeficiente de Reparabilidad ($C_{rep}$)**. Si un dispositivo no es reparable, su costo $R$ se multiplica, haciéndolo menos competitivo que uno modular.

## 7.3 Servicios y Cuidados
*   **Prioridad:** Tiempo ($T$) y Dignidad Humana.
*   **Justificación:** Sector intensivo en mano de obra. Riesgo de explotación y burnout.
*   **Ajuste V1.2:** Se protege el **Tiempo Opaco**. No se exige medir "output por hora" en cuidados (enfermería, educación), sino cumplimiento de SDV del trabajador.

        uint256 life;       // Vida (UCV * 100)
        int256 resources;   // Recursos (PIP * 100) - Puede ser negativo (Regenerativo)
        uint256 timestamp;  // Momento del registro
        string proofHash;   // Hash IPFS de la evidencia (IoT/Docs)
    }

    struct Agent {
        uint256 confidenceIndex; // IC (0-1000, donde 1000 = 1.0)
        bool isQuarantined;      // Estado de Cuarentena
        uint256 totalRegenerated; // Acumulador de R negativo
    }

    mapping(address => Agent) public agents;
    mapping(address => VectorVHV[]) public agentHistory;

    event VHVRecorded(address indexed agent, uint256 t, uint256 v, int256 r, uint256 ic);
    event QuarantineStatusChanged(address indexed agent, bool status);

    // Modificador: Solo permite registrar si no está en cuarentena (o si es auditor)
    modifier canRecord() {
        require(!agents[msg.sender].isQuarantined || isAuditor(msg.sender), "Agent in Quarantine");
        _;
    }

    // Función principal de registro
    function recordImpact(uint256 _t, uint256 _v, int256 _r, string memory _proof) public canRecord {
        
        // Registrar el vector
        agentHistory[msg.sender].push(VectorVHV({
            time: _t,
            life: _v,
            resources: _r,
            timestamp: block.timestamp,
            proofHash: _proof
        }));

        // Lógica simplificada de actualización de IC (en producción sería un Oráculo externo)
        if (_r < 0) {
            agents[msg.sender].totalRegenerated += uint256(-_r);
            // Bonificación por regeneración
            if (agents[msg.sender].confidenceIndex < 1000) {
                agents[msg.sender].confidenceIndex += 1;
            }
        }

        emit VHVRecorded(msg.sender, _t, _v, _r, agents[msg.sender].confidenceIndex);
    }
    
    // Función de lectura (pública y gratuita)
    function getLatestVector(address _agent) public view returns (VectorVHV memory) {
        require(agentHistory[_agent].length > 0, "No history");
        return agentHistory[_agent][agentHistory[_agent].length - 1];
    }
