设计为在传统市场中自主运作（作为彻底透明工具）或作为**马克苏克拉西**政治系统的会计引擎。

# 3. 基本原则（操作性公理）

该标准受到 6 个将真相公理翻译为工程规则的不可动摇原则的约束：

## 3.1 物理现实原则（反法定）
*不能通过法令创造价值。* 每份 VHV 报告必须有可验证的物理证据（传感器、日志、生物可追溯性）或直接社区共识的支持。金钱不是价值的证据。

## 3.2 事实/价值分离原则
*温度计不开处方。* EVV-1.2 测量物理量（`1000 升水`，`4 小时压力`）。它不决定这是"贵"还是"便宜"。估值（Maxo 中的定价）是社区随后使用自己的权重（$\alpha, \beta, \gamma, \delta$）执行的政治过程。

## 3.3 温柔隐私原则（V1.2 新增）
*效率不应杀死神秘。* 系统承认全面监视会破坏尊严。保证报告"不透明时间"或"照护时间"块的权利，无需详细列出活动，相信代理的 IC。

## 3.4 再生原则（V1.2 新增）
*目标不是零，而是正值。* 该标准设计用于测量和验证 R（资源）和 V（生命）组成部分中的负值，正式承认再生生态系统或愈合社会组织的行动。

## 3.5 继承责任原则
*没有人从零开始。* 每个产品都继承了使其成为可能的工具和基础设施的 VHV 的一部分。这使得依赖于先前肮脏采矿的"清洁"技术的隐藏成本可见。

## 3.6 公理演化原则
*地图不是领土。* 标准的技术参数（如排放因子或意识水平）通过科学共识和动态神谕验证定期更新，而非通过企业游说。

# 4. 数学模型：生命足迹向量（VHV）

VHV 不是单一数字（如价格），而是一个**三维向量**，保存伦理决策的关键信息。

$$VHV = [T, V, R]$$

其中每个分量在任何估值之前独立计算和标准化。

---

## 4.1 T 分量（时间）：生命的物质

测量人类**生命时间指数（TVI）**的消耗。承认时间是不可再生和存在性稀缺的资源。

### 通用公式
$$T = \sum (T_{direct} + T_{inherited} + T_{maintenance}) \times (1 - F_{opacity})$$

### 变量分解

1. **$T_{direct}$（有意识工作）：** 积极专注于创造或提供服务的人类生命小时数。
    * *V1.2 创新：* 区分"心流时间"（创造性/积极性）和"摩擦时间"（官僚/压力），尽管两者都计入总成本，第二个表示系统效率低下。

2. **$T_{inherited}$（时间背包）：** 结晶在所使用工具中的时间。
    * *计算：* `(工具创建_T / 估计寿命) × 使用时间`
    * *目的：* 避免"免费"自动化的幻觉。1 秒钟写文本的 AI 继承了数百万小时的人类训练和硬件开发。

3. **$T_{maintenance}$（未来债务）：** 社会将来必须投资以维护、修复或处置产品的时间。
    * *目的：* 惩罚一次性或难以修复的产品。

4. **$F_{opacity}$（隐私因子 - V1.2 新增）：** 一个系数（0.0 到 0.2），允许将某些时间声明为"私有/未审计"而不受 IC 惩罚，保护创造性或照护任务中的工人尊严。

### 测量单位
* **标准人小时（h·p）：** 1 小时有意识的成人生命。

---

## 4.2 V 分量（生命）：可量化伦理

测量对有意识生命的经验影响。用可验证的神经学量表替代模糊公式。

### 通用公式
$$V = \sum [Consciousness\_Level_i \times Duration_i \times Suffering\_Factor_i]$$

### A. 五个可验证意识层级（NC）
基于神经系统的复杂性和经历厌恶的能力（而非"灵魂"或主观"温柔"）。

| 层级 | 描述 | 因子（$NC$） | 示例 |
|------|------|-----------|------|
| **0** | **无中央意识** | 0.0 | 植物、真菌、细菌、矿物。（其保护落入 *R*）。 |
| **1** | **基本反应性** | 0.1 - 0.2 | 昆虫、双壳类。对刺激的反应，无明显情感记忆。 |
| **2** | **清晰意识** | 0.4 - 0.7 | 鱼类、爬行动物、家禽。可测量的学习、疼痛、压力。 |
| **3** | **复杂意识** | 0.8 - 0.95 | 哺乳动物、章鱼。情感、社会联系、游戏。 |
| **4** | **反思意识** | 1.0 | 人类（以及潜在的未来有感知 AI）。自我意识、时间投射。 |

### B. 苦难因子（FS）
关键乘数。这里是伦理所在。

* **FS = 0.2（有尊严的生命）：** 动物按其生物设计生活（SDV 满足）。*例如：放牧的母鸡。*
* **FS = 0.5（中度压力）：** 对自然行为的限制，但无急性疼痛。*例如：宽敞的牛棚。*
* **FS = 1.0（残忍/折磨）：** 系统性违反生物设计。*例如：笼养、群居物种的社会隔离、截肢。*
    * *V1.2 说明：* 在马克苏克拉西估值中，FS=1.0 触发 $\gamma$（痛苦）指数，使经济成本趋向无穷大。

### 测量单位
* **UCV（可验证意识单位）：** 相当于一个健康成年人一天的苦难。

---

## 4.3 R 分量（资源）：再生和限制

测量对星球生命支持的物理影响。引入再生逻辑。

### 通用公式
$$R = \sum [Quantity_i \times FRG_i \times (1 - C_{rep})] - R_{regenerated}$$

### 关键变量

1. **$FRG_i$（全球稀缺因子）：**
    * **1.0（关键/不可再生）：** 锂、金、大气（CO2 吸收能力）、濒危物种。
    * **0.5（慢速可再生）：** 可采伐森林、含水层、肥沃土壤。
    * **0.1（丰富/循环）：** 阳光、海水（用于非消耗性使用）、消费后回收材料。

2. **$C_{rep}$（可修复性系数 - V1.2 新增）：**
    * 0 到 1 之间的值，如果对象是模块化和可修复的，则降低 $R$ 成本。
    * *示例：* 用胶水密封的手机（$C_{rep} = 0$）支付全部资源成本。模块化手机（$C_{rep} = 0.8$）只支付材料成本的 20%，激励耐久设计。

3. **$R_{regenerated}$（再生积分 - V1.2 新增）：**
    * 允许负值。如果一个过程捕获的 CO2 多于排放的，或清洁的水多于污染的，则在该分量中生成**负 VHV**。这转化为生产者的"积分"或 Maxo。

### 基本测量单位
* **星球影响点（PIP）：** 标准化为 CO2 当量千克或按当地稀缺程度加权的升水。

## 5.2 信任指数（IC）：真相的货币

在去中心化系统中，声誉比金钱更有价值。**IC** 是分配给每个代理（个人或公司）的 0.0 到 1.0 之间的值。

### IC 状态机

IC 不是静态的；代理根据行为在状态之间"旅行"：

1. **已认证（$IC \ge 0.85$）：** *真相贵族。*
    * **特权：** 月度报告（非每日），间隔的随机审计，访问优惠 Maxo 信用。
    * **要求：** 12+ 个月的完美连贯性和净再生。

2. **合作（$0.70 \le IC < 0.85$）：** *标准公民。*
    * **特权：** 正常运营。其数据默认被接受为有效。
    * 通过之前审计的公司的**初始状态**。

3. **中性（$0.50 \le IC < 0.70$）：** *新人或可疑者。*
    * **条件：** 需要频繁的交叉验证。其报告自动应用预防"误差余量"。
    * 新个人的**初始状态**。

4. **风险（$0.30 \le IC < 0.50$）：** *黄色区。*
    * **原因：** 检测到不一致（例如，传感器说 X，报告说 Y）。
    * **后果：** 强制季度审计（由代理支付）。高价值交易被阻止。

5. **隔离（$IC < 0.30$）：** *临时公民死亡。*
    * **原因：** 经证明的故意欺诈或严重违反 SDV。
    * **后果：** 系统暂停。只能逐笔交易地进行外部验证运营。

### 动态调整算法
$$IC_{t+1} = IC_t + \Delta_{honesty} + \Delta_{regeneration} - (\Delta_{incoherence} \times Penalty)$$

* *注意：* 说谎的惩罚是说实话奖励的 3 倍。信任慢慢积累，迅速失去。

# 7. 按行业分类的参数（科学证据）

并非所有行业在生命天平上的权重相同。EVV-1.2 根据行业调整传感器灵敏度。

## 7.1 农业和食品
* **优先事项：** 生命（$V$）和资源（$R$）。
* **理由：** 农业负责 80% 的生物多样性丧失（Tillman 等）和 70% 的淡水使用。
* **V1.2 调整：** $\beta$（生命权重）提升至 **0.55**。应用高 $\gamma$（痛苦）**1.9** 来惩罚单一种植和动物禁闭。

## 7.2 技术和制造业
* **优先事项：** 资源（$R$）和继承时间（$T_{inherited}$）。
* **理由：** 密集使用稀土矿物（高 FRG）和计划性过时。
* **V1.2 调整：** 激活**可修复性系数（$C_{rep}$）**。如果设备不可修复，其 $R$ 成本乘以，使其相对于模块化产品竞争力下降。

## 7.3 服务和照护
* **优先事项：** 时间（$T$）和人类尊严。
* **理由：** 劳动密集型行业。剥削和倦怠风险。
* **V1.2 调整：** 保护**不透明时间**。不要求测量护理（护理、教育）中的"每小时产出"，而是要求工人满足 SDV。

## 智能合约 `VHVRegistry`（Solidity - V1.2）

这是记录向量的不可变核心。注意组件的分离和 IC 的包含。

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

/**
 * @title VHVRegistry V1.2
 * @dev 带有信任指数的生命足迹记录系统
 *      针对 gas 和隐私进行了优化。
 */
contract VHVRegistry {

    struct VectorVHV {
        uint256 time;       // 时间 (h·p * 100)
        uint256 life;       // 生命 (UCV * 100)
        int256 resources;   // 资源 (PIP * 100) - 可以为负（再生性）
        uint256 timestamp;  // 记录时刻
        string proofHash;   // 证据的 IPFS 哈希 (IoT/文档)
    }

    struct Agent {
        uint256 confidenceIndex; // IC (0-1000，其中 1000 = 1.0)
        bool isQuarantined;      // 隔离状态
        uint256 totalRegenerated; // 负 R 的累加器
    }

    mapping(address => Agent) public agents;
    mapping(address => VectorVHV[]) public agentHistory;

    event VHVRecorded(address indexed agent, uint256 t, uint256 v, int256 r, uint256 ic);
    event QuarantineStatusChanged(address indexed agent, bool status);

    // 修饰符：仅在不处于隔离状态时允许记录（或者如果是审计员）
    modifier canRecord() {
        require(!agents[msg.sender].isQuarantined || isAuditor(msg.sender), "Agent in Quarantine");
        _;
    }

    // 记录的主要函数
    function recordImpact(uint256 _t, uint256 _v, int256 _r, string memory _proof) public canRecord {

        // 记录向量
        agentHistory[msg.sender].push(VectorVHV({
            time: _t,
            life: _v,
            resources: _r,
            timestamp: block.timestamp,
            proofHash: _proof
        }));

        // 简化的 IC 更新逻辑（在生产中将是外部 Oracle）
        if (_r < 0) {
            agents[msg.sender].totalRegenerated += uint256(-_r);
            // 再生奖励
            if (agents[msg.sender].confidenceIndex < 1000) {
                agents[msg.sender].confidenceIndex += 1;
            }
        }

        emit VHVRecorded(msg.sender, _t, _v, _r, agents[msg.sender].confidenceIndex);
    }

    // 读取函数（公开且免费）
    function getLatestVector(address _agent) public view returns (VectorVHV memory) {
        require(agentHistory[_agent].length > 0, "No history");
        return agentHistory[_agent][agentHistory[_agent].length - 1];
    }

    // 治理逻辑的占位符
    function isAuditor(address _user) internal pure returns (bool) {
        return false; // 实施审计员白名单逻辑
    }
}
```
