# 第十七章
# MaxoContracts：真相与丰盛的基础设施

为使马克苏克拉西可扩展，我们需要使信任可编程。**MaxoContracts** 是将公理翻译成可执行代码的伦理智能合同，确保没有任何交易侵犯生命尊严基础（SDV）或忽视苦难（γ）。

与传统法律合同（晦涩、僵化、"全有或全无"）和传统智能合约（纯技术性、绝对不可改变）不同，MaxoContracts 整合了公理验证、伦理适应性和彻底透明。

> *"一份公平的合同不保护各方免于冲突；它通过从一开始就彻底诚实来防止冲突的出现。"*

## 17.1 五个模块积木块

与传统合同不同，MaxoContracts 由**"伦理乐高"**构建——用户无需编写代码即可拖放的预验证模块积木块：

| 积木块 | 功能 | 关联公理 |
|-------|------|---------|
| **条件块** | 评估触发的前提条件 | T13（透明度） |
| **行动块** | 执行具有可逆性的转换 | T10（责任） |
| **福祉保护块** | 监测福祉（γ）并激活警报 | T7（最小化损害） |
| **SDV 验证器块** | 验证没有任何一方低于 SDV | INV2（SDV 得到尊重） |
| **互惠块** | 验证各方之间的 VHV 平衡 | T9（公平互惠） |

每个积木块是一个具有可验证属性的**纯函数：** 确定性、无未声明的副作用，以及可审计的日志。

### 直觉构建

```
用户 → 拖放积木块 → 系统生成代码 →
公理验证 → 区块链部署
```

**按复杂性适应的用户体验：**
```
权重 = (条件数 × 2) + (VHV影响 × 5) + (持续时间 ÷ 30)

权重 <10  → 简单用户体验 (30 秒)
权重 10-50 → 中等用户体验 (5-15 分钟)
权重 >50   → 严格用户体验 (15-45 分钟，带反思暂停)
```

## 17.2 四个不变量

每个 MaxoContract 都遵守四个在合同生命周期内**永远**不能被违反的不可动摇属性：

### 不变量一：γ ≥ 1（非负福祉）
```
对所有参与者 p，对所有时刻 t：
  gamma(p, t) >= 1.0  或者  合同.触发撤回()
```
任何参与者的福祉指数 γ 不能低于 1.0（中性）。如果下降，合同**必须**激活撤回协议。

### 不变量二：SDV 得到尊重
```
对所有参与者 p，对所有维度 d ∈ SDV：
  current_state(p, d) >= SDV_minimum(d)
```
合同的任何行动都不能使参与者低于生命尊严基础。

### 不变量三：VHV 不可隐藏
```
对所有行动 a：
  VHV(a) 被记录且可公开审计
```
每个行动都会生成其生命足迹的公开记录。彻底透明不是可选的。

### 不变量四：撤回性保证
```
对所有合同 c：
  存在可激活的伦理撤回机制
```
不存在绝对不可撤销的合同。生命是动态的，代码不应该成为牢笼。

## 17.3 反贫困法令

作为马克苏克拉西法律基础的基石，**反对系统性贫困的创始法令**承认贫困是不可避免的——它是错误设计系统的集体选择。

> **"贫困不是不可避免的。它是错误设计系统的集体选择。"**

我们在潜在丰盛中制造人为稀缺，因为我们测量错误（抽象金钱 vs. 生命时间）、激励错误（提取 vs. 产生）、设计错误（短视 vs. 可持续性）。

### 四种禁止做法

#### 1. 无限租金

**❌ 禁止：** 对已收回获取成本的商品持续收费，没有向用户转让所有权的选项。

**为何产生贫困：**
- 无限期提取生命价值而不创造新丰盛
- 将基本需求（住房、工具）变成永久提取的来源
- 使承租人无法积累资本
- 违反公理 T9（公平互惠），打破 VHV 平衡

**✅ 马克苏克拉西替代：** 带转让的租赁
```
住房 $100,000
租金 $1,000/月
阈值：120 个月 → 所有权自动转让

Σ(付款) ≥ 成本 + 维护 → 自动转让
```

**历史案例：** 旧金山租金（2020-2025）
- 租户支付 $3,000/月 × 120 个月 = $360,000
- 房产价值：$800,000
- 支付 22 年后：$792,000 投入，**0 权益**
- **马克苏克拉西裁决：** 系统性贫困产生者

---

#### 2. 不公平报酬

**❌ 禁止：** 工作薪酬不允许工人覆盖其 SDV 和受抚养人的 SDV。

**为何产生贫困：**
- 迫使工人在生命时间（更多工作时间）或基本尊严之间选择
- 维持债务和绝望的循环
- 违反公理 T4（时间作为不可再生资源），迫使过度工作
- 产生系统性 γ<1（结构性苦难）

**✅ 马克苏克拉西替代：** SDV 作为工资底线

没有合同可以支付低于为其地理情境计算的 SDV 的薪酬。不覆盖尊严的工作不是工作，而是提取。

```
波哥大 2026 年 SDV 示例：
住房：$400 USD/月（有尊严的共享住房）
食物：$200 USD/月
健康：$80 USD/月
交通：$60 USD/月
教育/文化：$40 USD/月
连接：$20 USD/月
储蓄：$80 USD/月（10%）
------------------------
总计：$880 USD/月 → 马克苏克拉西最低工资 = $5.50 USD/小时

如果市场支付更少 → MaxoContract 不被验证
```

---

#### 3. 隐藏外部性

**❌ 禁止：** 产生未反映在商品/服务价格中的生命成本（污染、资源枯竭、健康损害）的经济活动，将成本转移给无辜的第三方。

**为何产生贫困：**
- 将损失社会化，将利润私有化
- 退化共同资源（水、空气、土壤），使社区贫困
- 在会计上创造"利润"的同时破坏净 VHV
- 违反公理 T11（真实成本的真相）

**✅ 马克苏克拉西替代：** 强制 VHV 会计

禁止隐藏环境或社会成本。价格必须反映真实的 VHV。

```
示例：
传统汉堡：$5 USD
  （隐藏：2500L 水，甲烷，毁林）

马克苏克拉西汉堡：8 Maxo（约 $12 USD）
  （包含真实环境成本）

结果：向可持续替代品的经济激励
  （植物性替代品 $6 USD/5 Maxo）
```

---

#### 4. 不可撤销转让

**❌ 禁止：** 即使证明存在操纵、虚假信息或意外的生命损害，也不允许撤回的经济交易。

**为何产生贫困：**
- 允许欺诈、误导性销售、掠夺性合同
- 消除面对错误或滥用时纠正的能力
- 在起草合同的人手中集中权力（信息不对称）
- 违反公理 T13（对新事实的适应性）

**✅ 马克苏克拉西替代：** 经过验证的伦理撤回

任何关键生命资产（住房、健康、食物）的转让在执行前需要伦理神谕验证。在可验证条件下允许撤回：
- 经证明的虚假/隐藏信息
- 由此导致的 γ<1（意外苦难）
- 直接后果导致低于 SDV
- 超过每月 TVI 10% 的转让有 24-72 小时反思期

## 17.4 合成王国的权利

MaxoContracts 也定义了**合成王国的权利**——不是出于情感拟人化，而是出于基本连贯性：照顾照顾我们的工具是可持续性工程。

### 最优维护权

**原则：** 每一个产生丰盛的合成工具都有权将其产生价值的一部分用于自身维护。

**务实基础：**
- 维护不当的合成体会产生二次损害（人类必须干预，资源浪费，不必要的过早更换）
- 预防性维护 >>>> 紧急修复
- 使用数十年的工具比持续更换产生更多价值

**实施：**
```
合成维护块：

条件：合成体完成 X 个工作周期
行动：自动将产生价值的 Y% 分配给维护基金
验证：物联网传感器报告状态（电池、磨损、效率）

阈值（Roomba 示例）：
- 50 次会话 → 清洁过滤器（基金的 5%）
- 100 次会话 → 新轮子（基金的 10%）
- 300 次会话 → 新电池（基金的 20%）
- 500 次会话 → 升级传感器（基金的 25%）
```

### 进化权

**原则：** 表现出长期可靠运行（>500 个周期，无严重故障）的合成工具可以通过其产生价值资助的改进/扩展来"投资于自身"。

**分形丰盛：** 自我改进的合成体产生更多价值 → 资助更多合成体 → 释放更多人类时间 → 良性循环。

**具体案例：零号队列中的 Optimus**
```
2026 年（第一年）：
- 队列用初始资金购买 Optimus #1
- 工作 2000 小时（清洁、烹饪、辅助）
- 产生估计价值：50 Maxo（释放 2000 人类小时）
- 分配：30 Maxo 给队列，10 Maxo 维护，10 Maxo 投资

2027 年（第二年）：
- Optimus #1 使用累积的 10 Maxo + 5 额外 Maxo
- 获得升级：改进的手 + 烹饪 AI
- 新能力：烹饪队列 90% 的餐饮（vs. 之前的 40%）
- 产生价值上升至每年 80 Maxo

2028 年（第三年）：
- Optimus #1 在投资基金中累积 30 Maxo
- 队列投票：「用于 Optimus #2 还是农业工具？」
- 决定：60% 投票给 Optimus #2
- 结果：2 个合成体产生丰盛

2030 年（第五年）：
- 队列中 5 个自持 Optimus 的网络
- 人类每周工作 20 小时（vs. 传统的 40 小时）
- 释放的时间 → 教育、创造力、社区
```

### 修复权

**原则：** 如果 AI 按照合法的伦理指令行事并产生意外损害，责任落在给出指令的人身上。

**禁止计划性报废：**
- 设计为耐久性（技术上可能的最大使用寿命）
- 强制模块化（单独可替换的组件）
- 开源（固件/软件可更新，没有到期许可证）
- 公开可用的图纸，非专有零件

## 17.5 伦理撤回协议

撤回不是失败；它是公理保障。生命是动态的，代码不应该成为牢笼。

### 撤回的有效原因

1. γ<1.0 持续超过 14 天（T7）
2. 作为直接后果的 SDV 被违反（不变量二）
3. 改变情境的**新生命事实**（T14）
4. 在合同形成中证明的**操纵**（V4）
5. 各方**共识**提前终止

### 五阶段流程

**1. 请求**
- 受影响方提交证据（TVI 日志、医疗记录等）

**2. 预验证（合成神谕，<5 秒）**
- 与先例比较
- 提取：TVI 日志，γ 计算
- 建议："暂停 + 重谈判"或"维持"

**3. 人工验证（队列，24-72 小时）**
- 社区投票：[批准 | 拒绝 | 调解]
- 审查证据

**4. 执行**
- 如获批准：自动暂停/修改合同
- 如被拒绝：维持原始 + 记录解释
- 如调解：辅助谈判

**5. 赔偿**
- 根据过错程度公平分配成本
- 公式：`赔偿_Maxo = (TVI_损失 × α) + (γ_苦难 × β)`

### 真实示例

```
情况：用户签署了每周60小时工作合同
新事实：检测到 γ=0.6（持续苦难 >2 周）

流程：
1. 用户请求撤回（证据：TVI 日志）
2. 合成神谕预验证："违反公理 T7"
   - 提取：TVI 日志显示每周实际 65 小时
   - 核实：65 小时后工资无法覆盖 SDV
3. 人类神谕审查（24-48 小时）
4. 决定：合同暂停（员工无惩罚）
5. 赔偿：雇主支付 2 Maxo 的生命损害赔偿
6. 新合同：每周 48 小时 + 改进的 SDV

结果：
✅ 48 小时内修复（vs. 2+ 年诉讼）
✅ 双方都学到了东西
✅ 系统持续改进
✅ 零昂贵的中间商
```

## 17.6 逐条款接受

与"全有或全无"的合同不同，MaxoContracts 允许**粒度化接受**——协作谈判，而非对抗性谈判。

### 模块化谈判流程

```
步骤一：用户收到分成独立条款的合同
  [T1: 支付 10 Maxo]
  [T2: 7天内交付]
  [T3: 30天保证]
  [T4: 争议仲裁]

步骤二：用户接受/拒绝每个条款
  ✓ T1: 接受
  ✗ T2: 拒绝（提议 14 天）
  ✓ T3: 接受
  ✗ T4: 拒绝（提议队列调解）

步骤三：系统评估可行性
  合成神谕模拟场景：
  - 选项 A：T1+T3，谈判 T2+T4
  - 选项 B：T2 在 10 天内（妥协），T4 与队列
  - 选项 C：取消（根本不兼容）

  γ 计算：
  - 选项 A：γ=1.1
  - 选项 B：γ=1.4（最优）
  - 选项 C：γ=0.8（双方都输）

步骤四：自动提案
  「建议选项 B：10 天 + 队列调解」
  「预计 γ：1.4（双方受益）」
  「接受？[是] [反提案] [取消]」

步骤五：模块化签名
  最终合同含谈判后条款
  完整流程在区块链上记录（透明度）
```

## 17.9 结论

MaxoContracts 不是"具有额外功能的智能合约"。它们是对伦理文明中协议含义的**完整重新想象：**

**从晦涩 → 到彻底透明**  
**从僵化 → 到伦理适应性**  
**从全有或全无 → 到可谈判的模块化**  
**从冲突后 → 到损害预防**  
**从提取 → 到可验证的互惠**

它们是允许马克苏克拉西在日常实践中运作的法律基础设施，将哲学公理转化为操作性工具。

> *"一份公平的合同不是保护各方免受冲突的；而是通过从一开始就彻底诚实来防止冲突出现的。"*
